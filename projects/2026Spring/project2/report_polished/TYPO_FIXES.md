# Typo 与格式问题清单（来自原稿 `report.md`）

## 1) 标题/空格规范

- `## 2.实验前的预期` → `## 2. 实验前预期`
- `### 3.1 macOS实验环境` → `### 3.1 macOS 实验环境`
- `## 6.实验方案设计` → `## 6. 实验方案设计`

## 2) 术语统一

- `check_sum` → `checksum`
- `signed char` / `signed_char`：正文建议统一为 `signed char`，CSV 字段保留 `signed_char`
- `win/mac` → `Windows/macOS`
- `avg_ns`、`total_ns`、`reps` 等变量名统一使用反引号包裹

## 3) 中英混排与标点

- 英文与中文之间补空格：如 `N较小` → `N 较小`
- `N=10^4时` → `N=10^4 时`
- `开启-O3` → `开启 -O3`

## 4) Markdown 结构

- 避免列表嵌套层级混乱：`6. - xxx` 这类写法应拆分为正常有序/无序列表
- 同一段不要混用多种强调风格（如全角引号 + 粗体 + 大段口语）
- 多图建议一图一段并附图注，避免一行堆叠 5 张图导致阅读拥挤

## 5) 文风建议

- 学术报告建议减少口语化表达（如“吓得瑟瑟发抖”）
- 结论段使用“观察—解释—边界条件”三段式，避免主观语气
